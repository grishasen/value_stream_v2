# Agent Instructions

- After testing with spawned Python processes, Streamlit servers, or other local dev servers, stop every process started for the test before finishing the turn.
- When a test server was bound to a port, verify the port is no longer listening before reporting completion.
- Before making architecture-sensitive changes, read `docs/index.md` and follow the engineer reading order in `docs/meta/documentation-guide.md`. Treat `docs/concepts/architecture.md`, `docs/concepts/domain-model.md`, and `docs/design/replacement-design.md` as the primary architecture sources; use `docs/reference/processors.md`, `docs/reference/algorithms.md`, `docs/reference/expression-dsl.md`, `docs/reference/readers-and-formats.md`, and `docs/reference/chart-catalog.md` for implementation details.
- Keep the core invariants visible while coding: Value Stream is aggregate-first, raw rows must not persist past chunk processing, YAML catalog configuration is the source of behavior, ingestion must remain deterministic and idempotent, persisted aggregates must carry provenance and config hashes, and query/UI paths should read through the aggregate/query layer rather than bypassing it. For behavior-changing work, update the relevant docs in the same change; the docs are the spec.
- Documentation is organized by the Diátaxis model (tutorials / guides / concepts / reference, plus design history under `docs/design/`); `docs/meta/documentation-guide.md` defines placement rules, content conventions, and the maintenance checklist. Significant design decisions get an ADR under `docs/concepts/adr/`.
